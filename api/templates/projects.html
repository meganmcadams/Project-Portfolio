Tuition Validation

Python/C++

I wrote a program in C++ to automate the tuition validation process for over 42,000 students and over half a billion dollars at the University of North Texas for the Student Accounting office.

It reads in student data, and using settings files, it determines how much each student should be receiving for a charge, if any. One settings file lists the restrictions for each charge using pseudocode to determine the conditions to receive it, and a second settings file lists the restrictions for each amount and how much per credit hour the amount costs as well as its cap. The original version was written in C++, but all of the settings were hardcoded. It was later rewritten in Python using dictionaries to store the data using external settings files, so no code needs to be changed when updating the settings. 

For my work on this project, I received the Outstanding Student Employee of the Year award in 2022 from UNT. 

Payment Plan Estimator

Python

I wrote a program in Python to estimate the division of a student's total tuition & fees for the semester across three installments with three different payment plan options: Short Term Loan, Installment Plan, and Installment Plan STB (with no down payment). This program has a functioning GUI to make it easier for the employees at Student Accounting to use. The GUI was built using PySimpleGUI. 

It reads in the student's total due charges and their anticipated aid (financial aid that has yet to disperse), reads in which payment plan the user would like to make an estimate for, subtracts the down payment (if any), and divides the remaining charges across the three payment deadlines. The anticipated aid is subtracted starting with the first installment and rolls over to other installments, as it doesn't evenly divide amongst the installments. 

Previous to this program, there was not an accurate way to estimate how installments would be broken up with anticipated aid for the Student Accounting office at UNT. Despite the simplicity of the program, it was used very often for students until we switched to a different payment plan method.

Tik Tak Toe

Python

When first learning Python, I made a simple Tik Tak Toe game using a list with each of the board slots and a list to hold each of the available slots.

The user can choose whether or not they want to go first and whether they are X's or O's, and when it is their turn, a board with numbers in the available slots is printed. The user types in a number, and their character (X or O) populates the slot. The bot picks a random available slot to occupy during its turn. A check is run at the end of each turn to determine whether or not there was a winner, and if so, who. The winner is stored in a save file so that the bot may react to seeing the same person again after the user reloads the program.

Discord Validation Bot

JavaScript

For my clubs' communication channels in Discord, I coded a bot to handle the verification of users. It is a UNT policy that official communication channels must have each member's name set to their first and last name, so I set up the channels so that when you first join, you are locked in the #verification channel until you obtain the @Verified role. 

The bot works by waiting for a checkmark reaction to the instructions for updating your nickname in the #verification channel, and it then compares the user's nickname (or username if no nickname is set) to a regular expression (^[A-Z][a-z]*+(([']?|[ ]?|[-]?)[A-z])*[ ][A-Z]([-][A-Z])?$) that checks if the user's nickname is a first name followed by a last initial. If it does not match, it messages the user explaining that they did not update their nickname accordingly and returns. If it does match, it gives the user the @Verified role and messages them with a success message.
